Blog Rendering Library

Zero-Dependency Vanilla JS · 15+ Custom Themes · MIT Open Source

Project Overview

The Blog Rendering Library is the engine that powers every blog post on Blogify. It is a standalone, open-source JavaScript library written entirely in vanilla JS with zero npm dependencies — no React, no Vue, no third-party parsers. Just pure JavaScript and direct DOM manipulation.

The library accepts a URL slug and theme name as query parameters, fetches the corresponding blog post JSON from the Blogify database, and renders it as fully styled semantic HTML in the browser — complete with typography, spacing, code blocks, images, and more — all controlled by the active theme.

With 15+ hand-crafted custom themes (from Tokyo Night to Dracula to Sepia to Neon), the library gives every Blogify blog post a distinct visual identity — switchable simply by changing a single URL query parameter.

How The Library Works

The library operates entirely in the browser — no server-side rendering, no build step. The rendering pipeline is triggered by URL query parameters, making theme-switching and content delivery completely URL-driven.

🔗
URL Parameters
?slug=post-id &theme=tokyo_night passed in the URL query string
→
📡
API Fetch
library_V2.js fetches JSON blog content from Blogify API using the slug
→
🎨
DOM Render
JSON blocks parsed → semantic HTML constructed via pure DOM manipulation → theme applied

The library reads the ?slug parameter to identify the blog post and calls the Blogify backend API to fetch its JSON content. It then reads the ?theme parameter, applies the matching theme's CSS variables and class system, and renders the full blog post by iterating over each JSON block — creating corresponding DOM elements for headings, paragraphs, images, code blocks, quotes, lists, and dividers — entirely through direct DOM manipulation with no framework involvement.

Key Features

  • Zero Dependencies: The entire library is a single library_V2.js file — no npm install, no node_modules, no bundler required. Just include it and it works.
  • JSON Block Rendering: Parses Blogify's structured JSON content format (headings, paragraphs, images, quotes, code, lists, dividers) into fully styled semantic HTML.
  • 15+ Custom Themes: Hand-crafted visual themes switchable via a single URL query parameter — Tokyo Night, Dracula, Nord, Solarized, Monokai, GitHub Light, Sepia, Neon, Aurora, Forest, Ocean, Midnight, Paper, Sunset, Minimal, and more.
  • URL-Parameter Driven: ?slug=post-id&theme=theme_name — completely URL-addressable renders. Share a link to any blog post in any theme.
  • Pure DOM Manipulation: No virtual DOM, no diffing algorithm, no framework overhead. Direct browser DOM API — lightweight and fast.
  • GitHub Actions Deployment: Automated CI/CD pipeline deploys to GitHub Pages on every push — zero manual deployment steps.
  • MIT Licensed: Open-source under MIT — free to use, fork, modify, and distribute. Co-credited to all three contributors.
  • Blogify Integration: Directly consumes Blogify's REST API using the slug as the key — tight integration with the parent platform's data format.
  • V2 Architecture: The current library_V2.js naming indicates a deliberate second version — improved over the initial implementation.
