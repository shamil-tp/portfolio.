LUME v2

Lightweight Video Streaming Platform

Project Overview

LUME is a modern, full-stack video streaming platform built entirely on the MERN stack. Designed as a lightweight but enterprise-grade alternative, it delivers seamless, secure video experiences through a highly responsive React frontend paired with an asynchronous, event-driven backend pipeline for video ingestion and processing.

Version 2 — the current iteration — moves far beyond simple file uploads, introducing a completely custom asynchronous video processing pipeline: Tus protocol for resumable uploads, a Redis-backed BullMQ message queue for background processing, local FFmpeg transcoding into HLS format, and zero-egress cloud storage via Cloudflare R2.

Video Processing Pipeline

LUME's core innovation is its fully asynchronous, microservice-style video pipeline — decoupling the upload, transcoding, storage, and playback stages into independent, fault-tolerant steps.

1
Ingestion — Resumable Chunked Upload
The React client uses Uppy to upload large raw video files to the Express server in chunks using the Tus protocol. If the network drops, the upload resumes exactly where it left off — no re-upload needed.
Tus Protocol · Uppy · tus-node-server
2
Queuing — Background Job Dispatch
Once the upload is fully assembled, the Express server logs video metadata to MongoDB and dispatches a transcoding job to the Redis queue via BullMQ — freeing the main API thread immediately.
BullMQ · ioredis · Redis · MongoDB
3
Processing — HLS Transcoding
An independent Node.js Worker process picks up the job and triggers FFmpeg via fluent-ffmpeg to transcode the raw video into .m3u8 playlists and .ts chunks for adaptive, buffer-free HLS streaming.
FFmpeg · fluent-ffmpeg · HLS · worker.js
4
Distribution — Cloud Storage Sync
The worker auto-uploads the entire HLS folder to Cloudflare R2 via the AWS S3 SDK for global content delivery with zero egress fees. MongoDB is updated with the public CDN URL. Local files are swept clean.
Cloudflare R2 · AWS S3 SDK · s3Uploader.js
5
Playback — Adaptive HLS Streaming
The custom React LumePlayer fetches the .m3u8 playlist from the CDN edge network and streams video chunks sequentially using hls.js — adaptive bitrate, buffer-free playback directly in the browser.
hls.js · LumePlayer.jsx · React

Key Features

  • Resumable Chunked Uploads: Tus protocol ensures massive video files survive unstable networks — upload pauses and resumes from the exact byte where it stopped.
  • Asynchronous Background Processing: Heavy video transcoding is offloaded from the main Express API to an independent worker process via BullMQ and Redis — the API stays responsive under load.
  • Native HLS Transcoding: FFmpeg converts raw video formats into .m3u8 playlists and .ts chunks — enabling adaptive, buffer-free streaming on any device.
  • Zero-Egress Cloud Storage: Processed HLS chunks are automatically synced to Cloudflare R2 via the AWS S3 SDK — global content delivery with no bandwidth costs.
  • Custom Adaptive Video Player: The LumePlayer component (React + hls.js) seamlessly stitches and streams HLS chunks directly in the browser — no plugin required.
  • Google OAuth2 Authentication: One-click sign-in via Google OAuth2 with JWT-based session persistence.
  • Video Discovery: DiscoverSection lets users explore all uploaded content; MyVideos gives a personal library view.
  • Automatic Local Cleanup: After cloud sync, the worker sweeps the local server clean — preventing disk bloat on the host machine.
  • TUS Upload Fix (documented): A dedicated TUS_UPLOAD_FIX_REPORT.md documents a critical upload bug investigation and resolution — showing production-grade debugging discipline.
  • Versioned Architecture: Two tagged releases — v1 (basic) and v2 (native streaming pipeline) — demonstrating systematic software evolution.
