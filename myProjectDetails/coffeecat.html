Project Overview

🎯
Hobby Project — Built for Pure Curiosity No client, no deadline, no team. Just shamil-tp exploring Vue.js at full depth — learning by building something real, complex, and actually fun to use on a phone.

CoffeeCat is a full-featured Instagram clone built entirely as a solo hobby project — driven by genuine curiosity and the desire to master Vue.js by building something real and non-trivial. It is not a tutorial copy. Every feature — real-time messaging, photo posting, likes, comments, follows, profile management, search, and full account deletion — was designed and implemented from scratch.

The project is deliberately mobile-only, primarily designed to run beautifully on a phone so it could be shown to friends without needing a laptop. This constraint shaped the entire UI/UX — it's designed the way Instagram actually feels on a phone screen.

Through CoffeeCat, shamil-tp went from Vue.js learner to Vue.js expert — mastering Vuex state management, Vue Router, Composition API, component architecture, real-time Socket.io bidirectional communication, Cloudinary media integration, and full MERN stack development at depth.

How The Project Works

CoffeeCat is a full MERN stack application — Vue.js SPA frontend communicating with an Express REST API backend, MongoDB for data persistence, Cloudinary for media storage, and Socket.io running a persistent WebSocket layer for real-time chat.

📱 Frontend — Vue 3 SPA (Mobile-First)
Vue Router (pages) Vuex (global state) Feed View Post Composer Profile Page Chat UI Search Users Socket.io client
↕ REST API (Axios + JWT)  |  WebSocket (Socket.io)
⚙ Backend — Node.js + Express REST API + Socket.io Server
Auth routes (JWT) Post routes User routes Chat routes Socket.io events Cloudinary upload Delete account pipeline
↕ Mongoose ODM  |  Cloudinary SDK
🗄 Data & Media Layer
MongoDB — users, posts, comments, follows, chats Cloudinary — photo storage + CDN

⚠ Thoughtful Feature — Account Deletion

Account deletion is not just a database flag — it is a full cascading cleanup pipeline. Deleting your account removes: your user document from MongoDB, all your posts and their Cloudinary images, all your comments, all your likes, and all your follow relationships. Chat messages are deliberately preserved for the person on the other side of the conversation — because their conversation history belongs to them too. This nuanced design decision shows real product thinking.

The Vue.js Mastery Journey

"I became an expert in Vue.js through this project."
CoffeeCat was not a guided tutorial — it was a self-directed deep dive. Building a full Instagram clone with real-time chat, Vuex state management, Vue Router, and Cloudinary integration forced mastery of Vue.js at every level — components, reactivity, lifecycle hooks, composables, stores, and the full ecosystem.

The progression through CoffeeCat covered every meaningful area of Vue.js development — from basic component authoring to complex Vuex store design, from simple route navigation to authenticated guards, from static templates to reactive real-time UI driven by Socket.io events. The mobile-only constraint meant every component had to be designed thoughtfully for touch interaction and small screen layouts — a discipline that sharpens frontend skills significantly.

My Contribution

CoffeeCat is a 100% solo project — every component, every API route, every Vuex store, every Socket.io event, every theme decision, every product feature was designed and built by shamil-tp alone. No contributors, no team. Pure solo exploration driven by curiosity.

  • Entire Vue.js Frontend: All components, all views, all Vuex modules, Vue Router configuration, mobile-first CSS — built from scratch.
  • Real-Time Chat System: Full Socket.io integration — server-side event handlers and client-side listeners for bidirectional live messaging.
  • Complete Backend API: All Express routes, controllers, and middleware for auth, posts, users, comments, likes, follows, chat, and media.
  • Cloudinary Pipeline: Photo upload on post creation, image deletion on post removal, full media cleanup on account deletion.
  • Account Deletion System: Designed and built the cascading delete pipeline — MongoDB multi-collection cleanup + Cloudinary purge + intentional chat message preservation.
  • Product Design: All UX decisions — the mobile-only constraint, the social features scope, the chat preservation policy — were made and implemented by shamil-tp alone.
