Osmium

Text Data Compression Program

Project Overview

Osmium is a data compression program built from the ground up for compressing text data. It is a server-rendered web application that takes raw text input, applies compression algorithms, and delivers the compressed output — all within a clean, minimal interface built with Node.js, Express, and EJS.

Named after one of the densest elements on the periodic table, Osmium reflects its core purpose: making data smaller and denser. This is a deep-dive into computer science fundamentals — implementing compression logic directly in JavaScript without relying on any third-party compression library.

How It Works

Osmium is a server-rendered MVC application. The user inputs text via the browser, the server processes it through custom compression logic in the utils/ layer, and the result is rendered back via EJS templates.

1
User Input: The user navigates to the web interface (served by Express) and enters raw text data into the input form.
2
HTTP Request: The form submits a POST request to the Express server with the raw text payload.
3
Compression Processing: The Express route calls the custom compression utility functions in the utils/ folder — applying the encoding algorithm to the input string.
4
File System Operations: Node.js fs and path modules are used to handle any file read/write operations during processing — reading input files or writing compressed output.
5
Rendered Output: The compressed result is passed to an EJS template, which renders the output back to the browser — showing the compressed data, size reduction stats, and encoding details.

Compression Approach

Osmium implements text compression using fundamental CS encoding techniques — built from scratch in JavaScript, without any external compression libraries. The utils/ folder contains the core algorithm implementations.

Run-Length Encoding (RLE) LIKELY CORE
One of the simplest lossless compression techniques — consecutive repeated characters are replaced by a count + character pair. Highly effective on repetitive text data.
e.g. "AAABBBCC" → "3A3B2C" — reduces character count significantly on repetitive strings
Huffman Coding LIKELY CORE
A frequency-based lossless compression algorithm that assigns shorter binary codes to more frequent characters and longer codes to rare ones — minimising total bit count.
e.g. 'e' (most frequent) → '0', 'z' (rare) → '11011' — variable-length prefix codes
Custom Utility Layer UTILS/
The utils/ folder contains the raw algorithm implementations — pure JavaScript functions for encoding and decoding text, abstracted cleanly from the Express routing layer.
Pure functions: encode(text) → compressed, decode(compressed) → original

Key Features

  • Text Compression Engine: Custom-built compression algorithm(s) implemented in pure JavaScript — no third-party compression libraries used.
  • Web Interface: Clean browser-based UI served by Express for submitting text and viewing compressed output without any CLI interaction.
  • Server-Side Rendering: EJS templates dynamically render input forms, compression results, and encoding details on the server.
  • File System Integration: Node.js fs and path modules handle file-based input/output — enabling file upload and compressed file download workflows.
  • Utility Layer Abstraction: All compression logic lives in the clean utils/ folder — fully decoupled from the Express routing and view layers.
  • Compression Metrics: Displays size reduction statistics — original size vs compressed size, giving users a clear view of efficiency.
  • Minimal Dependencies: Lean stack intentionally — the complexity is in the algorithm, not the framework layer.
