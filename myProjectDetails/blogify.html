Blogify

Collaborative Blogging Platform · OWASP 2025 Secured

Project Overview

Blogify is a full-stack collaborative blogging platform that lets users create, publish, and share rich blog content — powered by a custom-built blog rendering library written entirely in vanilla JavaScript with zero external dependencies. The editor experience mirrors WordPress's draggable block-based composition, giving writers a smooth, intuitive authoring flow.

What sets Blogify apart from typical CRUD blog apps is its enterprise-grade backend security architecture — built from scratch following OWASP 2025 guidelines, featuring layered defences including Helmet, HPP, rate limiting, Argon2 password hashing, Zod schema validation, and a dual access/refresh token system for secure API resource access.

Led by shamil-tp, built by a focused 3-person team, and live on Vercel — Blogify is a platform that takes both the writing experience and backend security as first-class concerns.

How The Project Works

Blogify is a decoupled full-stack platform — the React SPA frontend communicates with a hardened Express REST API backend, with all content serialised as structured JSON and rendered through a custom vanilla JS parsing library.

The blog editor serialises content as structured JSON blocks — headings, paragraphs, images, quotes, lists — which are stored in MongoDB as-is. On the reader side, the custom vanilla JS rendering library parses this JSON and constructs styled HTML through pure DOM manipulation, with zero npm dependencies.

Security Architecture

The backend security layer was entirely designed and implemented by shamil-tp, following the OWASP 2025 Top 10 guidelines to understand and apply real-world secure development practices. This is not boilerplate — every package has a specific threat it mitigates.

OWASP 2025 Guidelines Followed Backend designed against the OWASP Top 10 — injection, broken auth, security misconfiguration, rate limiting, and more. Real security research, not tutorial code.
HTTP Header Security
helmet ^8.1.0
Sets secure HTTP response headers — prevents clickjacking, MIME sniffing, XSS via CSP, and forces HTTPS via HSTS. First line of defence.
HTTP Param Pollution
hpp ^0.2.3
Prevents HTTP Parameter Pollution attacks — attackers sending duplicate query params to confuse middleware or bypass validation logic.
Rate Limiting
express-rate-limit ^8.2.1
Throttles requests per IP — protects against brute-force attacks on auth endpoints, credential stuffing, and API abuse/DoS attempts.
Schema Validation
zod ^4.3.6
Runtime schema validation on all incoming request bodies — rejects malformed, unexpected, or oversized payloads before they reach business logic.
Password Hashing
argon2 ^0.44.0
Argon2id — the winner of the Password Hashing Competition. Memory-hard algorithm, resistant to GPU brute-force and side-channel attacks. Stronger than bcrypt.
Dual Token Auth
jsonwebtoken ^9.0.3
Short-lived access tokens for API resource access + long-lived refresh tokens for silent renewal. Refresh token rotation prevents replay attacks and token theft.
Google OAuth2
google-auth-library ^10.5.0
Server-side Google ID token verification — no client-side trust. Validates token signature, audience, and expiry against Google's public keys.
CORS Policy
cors ^2.8.6
Strictly configured cross-origin resource sharing — only whitelisted origins can access the API. Prevents CSRF and unauthorised cross-origin API access.
Unique ID Generation
nanoid ^3.3.11
Cryptographically secure, URL-safe unique IDs — unpredictable resource identifiers to prevent enumeration attacks on blog and user resources.
SEO-Safe Slugs
slugify ^1.6.6
Sanitised URL slug generation for blog posts — normalises special characters, prevents path traversal or injection via crafted blog titles in URLs.

Custom Blog Rendering Library

Blogify's blog posts are not stored as raw HTML — they are serialised as structured JSON block arrays, and rendered on the client side by a custom-built vanilla JavaScript library that parses this JSON into styled, semantic HTML through pure DOM manipulation.

⚡ Coming Soon — Standalone Project
The Rendering Library
Built with zero npm dependencies — no React, no Vue, no third-party parsers. Pure vanilla JavaScript and direct DOM manipulation. The library takes a JSON block structure (headings, paragraphs, images, quotes, lists, code blocks) and constructs the full blog HTML in the browser.

Why build it from scratch? Full control over the render output, zero bundle weight overhead, and the ability to support custom Blogify-specific block types that no generic library could handle. The result is a fast, dependency-free rendering engine that powers every blog post on the platform.

This library is being extracted into its own standalone open-source project — watch this space. 👀

Key Features

  • Draggable Block Editor: WordPress-style drag-and-drop blog composition — reorderable content blocks including headings, paragraphs, images, quotes, lists, and code blocks.
  • Custom Rendering Library: Zero-dependency vanilla JS library parses structured JSON blog content into semantic, styled HTML — no React, no third-party parser. Pure DOM manipulation.
  • OWASP 2025 Security: Backend built from scratch following OWASP 2025 Top 10 — layered defences against injection, broken auth, security misconfiguration, and more.
  • Argon2id Password Hashing: PHC-winning memory-hard algorithm — significantly more secure than bcrypt, resistant to GPU-accelerated cracking.
  • Dual Token System: Short-lived access tokens + rotating refresh tokens — secure, silent session renewal without re-authentication.
  • Google OAuth2 (Server-side verified): ID token verified server-side via google-auth-library — no client-side trust. Signature and audience validated against Google's public keys.
  • Zod Request Validation: All incoming API payloads validated against strict Zod schemas — malformed, unexpected, or oversized data rejected before hitting business logic.
  • HTTP Parameter Pollution Protection: HPP middleware prevents attackers from sending duplicate query parameters to confuse middleware chains.
  • Rate Limiting: Per-IP request throttling on all endpoints — brute-force and credential stuffing attacks mitigated at the API layer.
  • Blog Themes: Multiple design themes for blog presentation — designed by Ranfees K, giving authors visual identity for their content.
  • SEO-Safe Blog URLs: Slugified, sanitised blog post URLs via nanoid + slugify — human-readable, collision-resistant, and injection-safe.
  • Collaborative Debugging: All three team members actively participated in debugging sessions — improving code quality and resilience across both frontend and backend.
